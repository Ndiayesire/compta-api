Introduction

L'API Insta Compta expose des endpoints REST sécurisés par JWT. Les réponses métier suivent l'enveloppe { success, message, data }. Les montants sont en Decimal (jamais en float) ; les dates en UTC côté serveur.

Base URL http://localhost:4230 en local
Format JSON · camelCase dans les corps HTTP
Auth Bearer JWT · Authorization: Bearer <token>
Postman Collection dans postman/ du dépôt

Démarrage rapide

1. Connexion

POST /auth/login est public. Récupérez accessToken et refreshToken.

POST /auth/login
Content-Type: application/json

{
  "email": "admin@example.com",
  "password": "votre-mot-de-passe"
}

2. Appeler un endpoint protégé

GET /clients
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

3. Rafraîchir le token

POST /auth/refresh
Content-Type: application/json

{ "refreshToken": "..." }

Authentification

Un guard JWT global protège la majorité des routes. Les routes publiques sont annotées côté serveur (ex. POST /auth/login, GET /health).

  • Access token — courte durée, à envoyer sur chaque requête.
  • Refresh token — renouvellement sans re-login.
  • Périmètre société — companyId issu du JWT filtre la plupart des données métier.

Conventions

JSON vs PostgreSQL

Les DTO HTTP restent en camelCase (clientId, countryId, …). Prisma mappe vers des colonnes PostgreSQL en snake_case (client_id, country_id).

Imports Excel

  • Multipart file (`.xlsx`, max 5 Mo)
  • clientId en query pour les imports par client
  • Max 500 lignes utiles par fichier
  • En-têtes ligne 1 — ordre libre, synonymes FR acceptés

Imports Excel

Plusieurs modules acceptent un import en masse. Les modèles sont dans src/assets/xlsx/. Réponse type : createdCount, failedCount, errors[].

Annexe fiscale TVA

La déclaration mensuelle s’appuie sur une annexe fiscale (lignes L5–L120). L’endpoint de calcul agège les opérations du mois puis applique les formules de solde. 

GET /tva-annexes/compute?clientId={{clientId}}&month=1&year=2025
Authorization: Bearer …

Paramètres optionnels :

  • reducedBase — L40 (part HT au taux réduit, défaut 0)
  • previousCredit — L100 (crédit reporté = L115 du mois précédent)
  • checksDdi — L75 (chèques DDI)
  • selfSupplies — L30 (informatif)
  • reducedRate — taux réduit en % (défaut 10)

Avant agrégation, chaque ligne opérationnelle complète le triplet total = net + tax (champ manquant déduit des deux autres). Bases d’assiette en HT (net) ; TVA collectée via L60 = L50 + L55. Taux normal : client.country.tva. Réponse : lines.L*, payable (L110), creditCarryForward (L115).

Formules L5–L115

# Complétion ligne : total = net + tax (champ manquant déduit)
L5   = Σ op_turnover_net + Σ op_exemption_amount   (= CA + L15)
L10  = Σ op_exportation_net
L15  = Σ op_exemption_amount
L20  = Σ op_suspension_net
L25  = L10 + L15 + L20
L35  = L5  − L25
L45  = L35 − L40
L50  = L40 × 10%
L55  = L45 × taux_pays%
L60  = L50 + L55
L80  = Σ op_importation_net
L85  = Σ taxDeduction (sinon tax) importations
L90  = Σ taxDeduction (sinon tax) achats locaux
L76  = L70 + L75
L91  = L85 + L90
L92  = L76 + L91
L105 = L70 + L75 + L85 + L90 + L100
L110 = max(0, L60 − L105)    → TVA à payer
L115 = max(0, L105 − L60)    → Crédit à reporter

Enchaînement mensuel : persister L115 pour alimenter L100 du mois suivant (query previousCredit). Arrondis à 2 décimales.
